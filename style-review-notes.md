# Style Review Feedback

## Key Improvements Needed

1. **Make the global visual system sharper** - More tactical HUD language, crisper panel framing, scanline/circuit/grid motifs, stronger distinction between passive surfaces, active systems, alerts, verified states. Glow should mean something.

2. **Increase typography authority** - Bigger confidence, stronger weight contrast, more monospace for operational values, IPs, domains, timestamps. SOC terminal + executive-grade polish.

3. **Landing page hero image** - Replace generic neon cybersecurity illustration with something showing actual email threat detection: envelope analysis, confidence scoring, branded scan logic.

4. **Tighten color semantics** - Analytics charts use rainbow colors that dilute signature palette. Keep red strictly for threat, green strictly for safe, cyan/indigo for intelligence/detection. Orange/yellow for risk severity only.

5. **Strengthen brand voice** - Replace generic phrases like "Get Started", "Feature coming soon" with decisive, technical, operational language for CISOs and SOC teams.

## Brief Amendments to Apply
- PhishGuard panels should use tactical HUD framing, circuit/grid accents, semantic neon edge states
- Major metrics, threat identifiers, domains, IPs, timestamps, confidence values should use monospace
- Electric Indigo #6C5CE7 and Cyber Cyan #00D2FF are dominant brand/intelligence colors
- Threat Red #FF3B3B only for malicious/phishing states
- Safe Green #00E676 only for verified/safe states
